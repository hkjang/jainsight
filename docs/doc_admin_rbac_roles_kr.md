# RBAC 역할 (RBAC Roles)

역할 기반 접근 제어(RBAC)를 통해 사용자 역할을 기반으로 세분화된 권한을 정의할 수 있습니다.

## 역할 유형 (Role Types)

- **시스템 역할 (빨간색)**: 내장된 변경 불가능한 역할 (예: Super Admin, Admin). 이는 핵심 시스템 기능에 대한 접근을 보장합니다.
- **사용자 정의 역할 (파란색)**: 특정 비즈니스 요구에 맞게 조정할 수 있는 사용자 정의 역할 (예: Data Analyst, Query Viewer).

## 역할 생성

- **이름 및 설명**: 역할의 목적을 명확히 식별합니다.
- **상위 역할**: 다른 역할로부터 권한을 상속받습니다.
- **우선순위**: 충돌 해결 우선순위(1-100)를 정의합니다. 숫자가 높을수록 우선순위가 높습니다.

## 대시보드

RBAC 대시보드는 다음 요약 정보를 제공합니다:

- 총 역할 수.
- 시스템 역할 대 사용자 정의 역할의 분포.
- 이러한 역할에 할당된 총 사용자 수.
