# 보안 설정 (Security Settings)

AI 및 데이터 플랫폼에 대한 전역 보안 매개변수를 구성합니다.

## 보안 점수 (Security Score)

현재 구성을 기반으로 한 실시간 보안 태세 평가(0-100)입니다. 인젝션 검사 및 PII 마스킹과 같은 기능을 활성화하면 점수가 올라갑니다.

## 보호 계층

1.  **인젝션 방어**:
    - **프롬프트 인젝션**: AI의 지시를 조작하려는 시도를 탐지합니다.
    - **SQL 인젝션**: 생성된 SQL에서 악의적인 패턴을 스캔합니다.
2.  **구문 제어**:
    - **DDL 차단**: 스키마 변경을 전역적으로 방지합니다.
    - **DML 차단**: 데이터 변경을 전역적으로 방지합니다.
    - **키워드 차단 목록**: 특정 금지 단어(예: `GRANT`, `REVOKE`)를 정의합니다.
3.  **데이터 프라이버시 (PII)**:
    - **컬럼 패턴**: 자동으로 민감한 정보로 처리할 컬럼에 대한 정규식/패턴을 정의합니다(예: `*password*`, `*ssn*`).
4.  **운영 제한**:
    - **속도 제한 (Rate Limiting)**: 분당 최대 요청 수를 설정합니다.
    - **감사**: 보안 로그에 대한 보존 정책 설정.

## 보안 이벤트

"DDL 차단됨" 또는 "인젝션 시도 방지됨"과 같은 최근 보안 개입에 대한 실시간 피드로, 위협에 대한 즉각적인 가시성을 제공합니다.
