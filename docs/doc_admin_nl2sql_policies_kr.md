# NL2SQL 정책 (NL2SQL Policies)

자연어가 SQL로 변환되는 방식을 제어하는 정교한 규칙을 정의합니다.

## 정책 제어

- **DDL/DML 차단**: AI가 생성한 `CREATE`/`DROP` (DDL) 또는 `INSERT`/`UPDATE` (DML) 문을 차단하는 스위치.
- **인젝션 검사**: AI 기반 분석을 활성화하여 프롬프트 인젝션 시도를 탐지합니다.
- **PII 마스킹**: 결과에서 민감한 컬럼(주민번호, 신용카드 등)을 자동으로 마스킹합니다.
- **행 제한**: 데이터 덤프를 방지하기 위해 최대 행 수(예: 1,000행)를 강제합니다.

## 프리셋 (Presets)

표준 보안 태세를 빠르게 적용합니다:

- **읽기 전용 (보수적)**: 모든 변경을 차단하고 엄격한 마스킹을 적용합니다.
- **분석가 (표준)**: 더 많은 행을 허용하되 DDL은 계속 차단합니다.
- **개발자 (유연)**: 개발 스키마에 대해 DDL/DML을 허용합니다.

## 비교 도구

두 개의 서로 다른 정책을 선택하여 권한 및 제약 조건의 차이점을 강조하는 비교 테이블을 나란히 확인합니다.
