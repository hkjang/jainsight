# 쿼리 실행 및 관리

효율적인 데이터 분석을 위해서는 쿼리 실행 방법을 마스터하는 것이 필수적입니다.

## 쿼리 작성 (Writing Queries)

- **자동 완성**: 타이핑 시 연결된 스키마를 기반으로 SQL 키워드, 테이블명, 컬럼명을 자동으로 제안합니다.
- **구문 강조**: 키워드, 문자열, 주석이 색상으로 구분되어 가독성을 높입니다.
- **다중 쿼리**: 세미콜론(`;`)으로 구분하여 여러 쿼리를 작성할 수 있습니다. `모두 실행(Execute All)`을 사용하면 순차적으로 실행됩니다.

## 실행 컨트롤 (Execution Controls)

1.  **실행 (Ctrl+Enter)**: 현재 선택된 쿼리 또는 커서가 위치한 쿼리를 실행합니다.
2.  **취소 (Cancel)**: 쿼리 실행이 너무 오래 걸릴 경우 "취소" 버튼이 나타나 요청을 중단할 수 있습니다.
3.  **실행 계획 (Explain)**: `EXPLAIN ANALYZE`를 사용하여 쿼리 실행 계획을 확인하고 성능 비용을 이해할 수 있습니다.

## 상태 피드백

- **성공**: 실행 시간(예: "성공 (12ms)")과 함께 녹색 알림이 나타납니다.
- **오류**: 데이터베이스 드라이버에서 반환된 구체적인 오류 메시지가 빨간색 경고 박스에 표시되며, 종종 해결책 제안이 함께 제공됩니다.
