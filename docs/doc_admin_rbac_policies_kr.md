# RBAC 정책 및 시뮬레이션 (RBAC Policies & Simulation)

접근을 제어하는 구체적인 규칙을 관리하고 그 적용 결과를 검증합니다.

## 권한 (정책)

각 역할에 대해 권한 목록을 정의할 수 있습니다:

- **범위 (Scope)**: 권한의 도메인 (System, Database, Schema, Table, Query).
- **리소스 (Resource)**: 대상 식별자. 와일드카드를 지원합니다 (예: `db:production:*`).
- **액션 (Action)**: 무엇이 허용되는가? (`read`, `execute`, `modify`, `delete`, `admin`).
- **효과 (Effect)**: **허용 (Allow)** (초록색) 또는 **거부 (Deny)** (빨간색). 명시적 거부는 허용보다 우선합니다.

## 시뮬레이션 도구

**시뮬레이션** 탭을 통해 관리자는 실제 사용자에게 영향을 주지 않고 권한 검사를 테스트해 볼 수 있습니다.

- **입력**: 사용자 ID, 리소스 문자열, 액션.
- **출력**: 명확한 `접근 허용` 또는 `접근 거부` 결과.
- **근거**: 어떤 권한 규칙이 매칭되어 해당 결과가 나왔는지 표시합니다 (예: "Role 'Admin' grants read access to database:\*").

## 매트릭스 및 템플릿

- **매트릭스**: (예정) 여러 역할 간의 권한을 비교하는 시각적 그리드.
- **템플릿**: (예정) 빠른 적용을 위한 재사용 가능한 정책 세트.
