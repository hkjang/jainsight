# 쿼리 정책 (Query Policies)

SQL 쿼리 실행 방식을 제어하고 조직 전반에 보안 표준을 시행합니다.

## 정책 관리

위험한 쿼리를 자동으로 감지하고 처리하는 규칙을 정의합니다.

- **정책 유형**:
  - **DDL 차단**: `DROP`이나 `ALTER`와 같은 구조적 변경을 방지합니다.
  - **WHERE 필수**: `DELETE`/`UPDATE` 문에서 `WHERE` 절 사용을 강제합니다.
  - **Limit 필수**: 결과 집합의 크기를 제한합니다.
- **조치 (Actions)**:
  - **차단 (Block)**: 쿼리 실행을 즉시 중단합니다.
  - **경고 (Warn)**: 실행은 허용하되 위험으로 표시합니다.
  - **감사 (Audit)**: 검토를 위해 조용히 로깅합니다.
- **위험도 점수**: 각 정책에 위험도 가중치(0-100)를 부여합니다.

## 실행 이력 (Execution Log)

쿼리 실행 이력을 검토합니다.

- **상태**: `Allowed` (초록), `Blocked` (빨강), `Warned` (주황).
- **위험 분석**: 각 쿼리에 대해 계산된 위험도 점수를 확인합니다.
- **차단 사유**: 쿼리 실행이 방지된 정확한 이유를 보여줍니다.

## 테스트 플레이그라운드

정책을 적용하기 전에 안전하게 검증하세요.

- **입력**: SQL 쿼리를 테스트 영역에 붙여넣습니다.
- **출력**: 시스템이 정책 검사를 시뮬레이션하여 "실행 가능" 또는 "정책 위반" 결과와 함께 위험도 점수 및 트리거된 규칙을 반환합니다.
